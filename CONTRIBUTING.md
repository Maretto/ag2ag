# Contributing to ag2ag

Thank you for your interest in contributing! We welcome bug reports, feature requests, and pull requests.

## Setup Instructions

1. Clone the repository:
   ```bash
   git clone https://github.com/Maretto/ag2ag.git
   cd ag2ag
   ```

2. Install dependencies:
   ```bash
   npm install
   ```

3. Run tests to ensure everything is working:
   ```bash
   npm test
   ```

## Pull Request Process

1. **Fork** the repository and create your branch from `main`.
2. **Branch** naming convention: `feature/your-feature` or `bugfix/issue-number`.
3. **Test** your changes using `npm test`. Make sure all tests pass and consider adding new tests for your features.
4. **Submit** your Pull Request with a clear description of the problem and the proposed solution.

## Code Style

- Please follow the existing patterns and conventions in the codebase.
- We prefer keeping dependencies to an absolute minimum (native Node.js modules when possible).
- Keep code clean, readable, and well-documented.

Thanks for helping make ag2ag better!
